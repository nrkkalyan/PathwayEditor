Installing yFiles

Prerequisites

To build applications that are based on yFiles Complete or yFiles Viewer you need to have a Java2 Software Development Kit version 1.4 or higher installed on your system. To build applications that are based on yFiles Layout or yFiles Basic it suffices to have a Java2 Software Development Kit version 1.3 installed on your system. We suggest to use J2SE 1.5 SDK or higher from Sun Microsystems which is freely downloadable at http://java.sun.com/j2se for Microsoft Windows, Linux, and Solaris operating systems.

If you just want to run the provided demo applications it suffices to have a Java2 Runtime Environment version 1.4 or higher installed. We suggest to use the J2SE 5 from Sun Microsystems which is freely downloadable at http://java.sun.com/j2se for Microsoft Windows, Linux, and Solaris operating systems.

Then you need a program to unpack the yFiles distribution. For that you can use Winzip for Windows. If you are on a UNIX platform use the unzip command. Alternatively, you can use the jar command that is part of the JSDK distribution.

Installation

Unpack the yFiles Zip archive to a directory of your choice. The yFiles distribution will be contained in a directory prefixed yfiles.

In order to use the yFiles library you need to include the Jar file y.jar in your Java classpath.

Optionally, you should download the yFiles Developer's Guide. It can be obtained from the same source as the yFiles class library itself. The Developer's Guide comes as a zipped archive file. To install it, simply unpack the archive to the yFiles documentation directory <yFilesDir>/doc so that the links contained in the documentation work.

Note: During installation you may be asked by your unzip program whether or not to overwrite the file developers-guide/index.html. In that case answer "Yes."

That's basically it.

You can verify that the installation was correct by running the demo programs that are part of the yFiles distribution.

Note that the resource files located in the demo subfolders or the tutorial subfolder need to be in your classpath in order for the demos to properly function. IDEA users should add at least the resource patterns ?*.png;?*.jpg:?*.gif:?*.gml;?*.ygf;?*.graphml;?*.xml;?*.xsl;?*.csv to their compiler settings.
